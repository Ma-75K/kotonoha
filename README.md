# ことのは

## サービス概要
忙しい子育ての中で、
「だっこ」「ぶーぶ」「できた！」といった
今しか聞けない言葉を前に、
「この言葉、忘れたくない」と思いながらも、
記録できずに過ぎてしまう瞬間があります。
「ことのは」は、子どもの声と言葉をワンタップで残し、
未来の宝物に変える、子育て向け記録アプリです。

- 子どもの今だけの可愛い言葉や言い間違いを残したくても忙しくて記録できないパパママに
- 思い出を「今すぐ、無理なく、そのままの形で」残せる体験を
- 子どもの「言葉」と「声」を記録する、未来に残る思い出保存アプリ

## メインのターゲットユーザー
　年齢：25～40歳
　属性：0～5歳の子どもを育てている親
　状況：初めての子育て/日記や記録が続かない/スマホ操作は問題なくできる
　
## 解決したい課題
- 忙しくて記録する余裕がない
- 残したい気持ちはあるのに、残せていない
- 思いついた瞬間はスマホ操作が難しい
- 日記形式は時間と手間がかかる、そして続かない

## 提供する価値
- 【今すぐ残せる】:思い立った瞬間にワンタップで記録
- 【そのまま残せる】:声・言葉・雰囲気を含めて保存
- 【後から振り返れる】:「〇年前の今日」で思い出を再発見
- 「記録したい瞬間と」「記録できる状態」のズレを解消

## 主な機能
- 音声録音による言葉の記録（起動→録音→保存だけの最短導線/後から編集可能）
- テキスト入力・編集
- 年齢/日付の自動付与（3歳2か月など）
- 複数の子どもの記録管理(兄弟対応)
   ∟ タブで切り替え、記録時に自動選択
- 記録の一覧表示（日/月/年ごとに「言葉の成長」が並ぶ）
- 音声再生
- 「〇年前の今日」を振り返る機能

## 技術スタック
- バックエンド: Ruby on Rails
- データベース: PostgreSQL(本番) / SQLite(開発)
- ストレージ: ActiveStorage(音声保存)
- フロントエンド: Rails標準(ERB) / Hotwire(Turbo+Stimulus) / Web Audio API(録音)
- デプロイ: Render / Fly.io / Railway(いずれか)

## なぜこのサービスを作りたいのか

子どもの成長の中で、ふとした瞬間に心に残る言葉があります。

「この言葉、忘れたくない」

そう思ってはいても、育児や日常の忙しさの中で書き留められず、時間が経つにつれて、その声やその時期特有の言い方を思い出せなくなってしまったという経験がありました。

写真や動画は自然と増えていく一方ですが、子どもの言葉そのものを残す手段は少ないと感じました。

既存の育児系アプリは、成長管理や記録の継続を目的としたものが多く、「残したい瞬間にだけ使える」シンプルな体験に特化したものは少ない印象です。

そこで、毎日続けることを前提にせず、完璧な記録を求めなくてもいい、ただ思い立ったその瞬間に、子どもの「声」と「言葉」をそのまま残せるサービスを作りたいと思い、本アプリを考えました。

優先度：高（MVP）
- [ ] ユーザー登録・ログイン機能
- [ ] 子ども情報の登録(名前・生年月日)
- [ ] 音声録音機能(Web Audio API)
- [ ] 音声ファイルの保存(Active Storage)
- [ ] テキスト入力による記録
- [ ] 記録一覧表示(日付順)
- [ ] 子ども別フィルター(タブ切り替え)
- [ ] 音声再生機能
- [ ] 「〇年前の今日」表示

優先度：中（体験向上）
- [ ] 音声の自動文字起こし
　　　　∟ 録音した音声をテキストとして変換して残す
- [ ] タグ付け/検索機能
     　∟ 成長やシーン別に記録を整理し、
     　　シーン・感情タグ/検索で「あのときのあのワード」だけ振り返られる機能
- [ ] 表示UIの改善
　     ∟ 操作回数を減らし、直感的に使える設計

優先度：低（将来的な拡張）
- [ ] 家族間での記録共有
　     ∟ 両親・祖父母と記録を共有できる

- [ ] データエクスポート機能
　     ∟ 将来、別の形で保存できる仕組み

## 既存サービスとの差別化
- 育児管理アプリ: 管理項目が多く、気軽に残しづらい
- メモ・音声メモアプリ: 用途が広すぎて、後から目的の記録を探しづらい
- 写真アプリ: 言葉そのものが主体になりにくい

「ことのは」は、「子どもの言葉を残す」ことだけに特化し、
ワンタップで記録し、思い出として振り返れる体験を提供します。
